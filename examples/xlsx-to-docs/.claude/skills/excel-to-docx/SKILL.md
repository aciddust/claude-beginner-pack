# 엑셀 → docx 일괄 변환

## 입력

- 작업 폴더에 다음이 있다고 가정:
  - 엑셀 파일 1개 (`*.xlsx`) — 데이터
  - docx 템플릿 1개 (`*.docx`) — `{{컬럼명}}` 자리 표시자 포함
- 사용자가 어느 파일이 데이터고 어느 게 템플릿인지 명시하지 않으면 일단 추측하고 확인 받는다.
  (보통 파일명에 "데이터", "템플릿" 같은 힌트가 있음)

## 절차

1. 엑셀의 첫 번째 시트를 읽고, 첫 행을 헤더로 사용한다.
2. 헤더 컬럼명들을 키, 각 행을 한 명의 데이터로 만든 dict 리스트 구성.
3. 템플릿 docx 를 한 번 읽어 메모리에 저장.
4. 각 사람마다:
   - 템플릿 사본을 만든다.
   - 본문/표/머리글/바닥글 안의 모든 `{{컬럼명}}` 을 해당 값으로 치환. (숫자·날짜 등 비문자열 값은 `str()` 로 변환 후 치환)
   - 출력 파일명: `결과/<이름>_<원본템플릿이름>` (예: `결과/김철수_템플릿.docx`)
   - "이름" 컬럼이 없으면 행 번호로 대체.
5. 진행 상황을 표시. 완료 후 생성된 파일 수와 실패 항목을 정리해 보고.

## 사용 라이브러리

- openpyxl — 엑셀 읽기
- python-docx — docx 읽고 쓰기

## 주의

- `결과/` 폴더가 없으면 먼저 만든다.
- 같은 이름의 출력 파일이 이미 있으면 사용자에게 덮어쓰기 여부를 묻는다.
- 데이터 행이 없거나 헤더가 비어있으면 즉시 사용자에게 알리고 중단.
- **개인정보 주의**: 실행 결과 파일은 사내 보안 정책에 따라 처리. 외부 송출 X.

## 예시 호출

사용자가 작업 폴더에서 다음과 같이 부탁:

```
이 엑셀 데이터로 평가서 docx 일괄 생성해줘
```

→ 자동으로 위 절차 수행.
